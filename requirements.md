# requirements.md
_最終更新：2026-04-18_

---

## 1. 目的

Personal Log Hubは、日々の短い記録をカテゴリ付きで保存し、あとから一覧で振り返るための最小ツールである。

フェーズ1では、入力と一覧確認に必要な最小機能に絞り、GASを保存先として利用する。

---

## 2. 対象ユーザー

| ユーザー層 | 使い方 |
|-----------|--------|
| 個人ユーザー | 日々の出来事やメモを短く記録する |
| 学習者 | 学習ログをカテゴリ付きで残す |
| 作業者 | 仕事や生活の簡易ログを一覧で確認する |

---

## 3. 対象スコープ（フェーズ1）

- 入力画面でカテゴリとテキストを送信する
- 一覧画面で保存済み記録を取得する
- 一覧画面でカテゴリフィルター表示する
- GAS Web Appと通信する
- GitHub Pagesへ自動デプロイする

---

## 4. 技術構成

| 項目 | 内容 |
|------|------|
| フロントエンド | Vite + React |
| 通信 | GETは`fetch`、POSTはhidden form submit |
| GAS URL | `VITE_GAS_URL` |
| 保存先 | GAS Script Properties |
| ホスティング | GitHub Pages |
| CI/CD | GitHub Actions |

---

## 5. 成功条件

- 入力画面からカテゴリ付きテキストをPOSTできること
- 一覧画面でGASから記録をGETできること
- カテゴリフィルターで表示対象を切り替えられること
- `VITE_GAS_URL` 未設定時に画面でエラーを表示すること
- GitHub Pages用のビルドとデプロイworkflowが配置されていること
